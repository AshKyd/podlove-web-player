Podlove Web Player v4 - Architektur Prototyp

Architektur

  • Basis: Redux Store
  • Media Treiber: Mediaelement Howler
    • Mediaelement Howler ohne Player
    • API treiben store
    • Kann durch belibige andere Mediaelement Bibliotheken ausgetauscht werden
    • Mediaelement über Store entkoppelt vom UI
  • UI Renderer: react
    • Austauschbar (bspw. Vue.js)
    • Integriert sich gut mit Redux Store
    • Viele Bibliotheken für Theming, i18n, ...
    • UI über Store entkoppelt von Mediaelement
  • Actions
    • Abstraktion für den Store
    • Gewährleistet Austauschbarkeit von Action Types

Einschränkungen vom Prototyp

  • Tabs fehlen (Kapitel, Transkripte, Einstellungen)
  • Kein Theming (mehrere Möglichkeiten, bspw. fixierte Themes wie in Material UI)
  • Kein i18n (bspw. auf Basis von react-intl)
  • Kein Depplinking (kann relativ simpel als redux store akteur implementiert werden)
  • Nur ein player (mehrere Lösungsansätze: ein store pro player oder ein store pro seite)
